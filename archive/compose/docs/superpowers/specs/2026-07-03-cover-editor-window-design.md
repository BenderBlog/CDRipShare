# 独立封面编辑窗口设计

## 背景

当前主窗口右侧的 `ImagePickerPanel` 同时承担封面选择、原图缩略图、生成后预览、背景模式下拉和自定义颜色入口。随着封面生成能力增加，这块区域已经偏拥挤，也让“封面编辑”看起来像一个附属小选项，而不是独立工作流。

本次设计将封面编辑能力拆到一个独立 Compose Desktop 窗口中。主窗口保持总览和合成入口，封面窗口专门处理封面选择与背景设置。

## 目标

- 主窗口右侧封面区域更轻，只展示当前封面状态和进入编辑器的入口。
- 提供一个真正独立的“编辑封面”窗口，而不是主窗口模态弹窗。
- 不引入草稿态；封面设置实时生效，主窗口和编辑窗口共享同一份状态。
- 保留现有背景模式、自定义颜色、封面预览和封面生成逻辑。

## 非目标

- 不新增封面裁剪、缩放、阴影、圆角、文字叠加等更深的版式编辑能力。
- 不做“保存/取消”草稿流程。
- 不改变合成视频的文件生成流程。
- 不引入新的 UI 框架或窗口管理库。

## 用户体验

### 主窗口

主窗口右侧封面区保留：

- 原始封面缩略图。
- 当前封面文件名和图片尺寸。
- 生成后的 16:9 封面预览。
- “编辑封面”按钮。

背景模式下拉、自定义颜色入口和调色板入口从主窗口移出，放到编辑窗口中。

点击“编辑封面”会打开封面编辑窗口。重复点击不会创建多个窗口，只会保持同一个编辑窗口处于打开状态。

首版不强制实现“已打开窗口主动抢焦点/置顶”。如果编辑窗口已经打开但被其他窗口遮住，重复点击主窗口按钮不会创建第二个窗口；后续如果需要更强的找回窗口体验，再单独增加焦点管理。

### 编辑封面窗口

编辑窗口建议初始尺寸为 `760dp × 560dp`，最小尺寸为 `680dp × 480dp`。窗口标题为 `编辑封面`。

窗口内容分为两侧：

- 左侧：大号 16:9 生成封面预览。未生成预览或未选择封面时显示空态。
- 右侧：封面设置控件，包括选择/更换封面、背景模式、自定义颜色状态和打开调色板按钮。

编辑窗口关闭时只隐藏窗口，不清空封面文件、背景模式、自定义颜色或预览。

## 状态模型

封面编辑窗口和主窗口共享同一个 `MainViewModel`。

继续使用现有状态：

- `imageFile`
- `coverPreview`
- `bgMode`
- `customColorHex`
- `onBgModeChanged()`
- `setImageFile(...)`

新增窗口显示状态：

- `showCoverEditor` 放在 `Main.kt` 的 `application {}` 作用域内。

不把窗口显示状态放进 `MainViewModel`，因为它是桌面窗口生命周期状态，不是业务数据。

## 架构

`Main.kt` 负责两个顶层窗口：

```kotlin
application {
    val viewModel = MainViewModel()
    var showCoverEditor by remember { mutableStateOf(false) }

    Window(...) {
        App(
            viewModel = viewModel,
            onOpenCoverEditor = { showCoverEditor = true }
        )
    }

    if (showCoverEditor) {
        Window(
            onCloseRequest = { showCoverEditor = false },
            title = "编辑封面",
            state = rememberWindowState(width = 760.dp, height = 560.dp)
        ) {
            CoverEditorWindow(...)
        }
    }
}
```

`App.kt` 增加 `onOpenCoverEditor` 参数，并将它传给右侧封面概览组件。

`ImagePickerPanel.kt` 拆分职责：

- 主窗口使用新的轻量封面概览组件。
- 编辑窗口使用新的封面编辑组件。
- 现有颜色选择器和背景模式选择器改为可复用的私有或内部组件，避免复制逻辑。

建议新增文件：

- `src/main/kotlin/io/github/benderblog/cdripshare/ui/CoverEditorWindow.kt`

## 交互规则

- 未合成时，主窗口和编辑窗口都可以操作其他非冲突控件。
- 修改背景模式或自定义颜色后，立即调用 `onBgModeChanged()` 重新生成预览。
- 更换封面图片后，立即更新 `imageFile` 并重新生成预览。
- 合成中，封面编辑控件禁用，但预览仍可查看。
- 点击“开始合成”时使用当前最新封面设置。

## 错误处理

- 如果封面图片读取失败，主窗口和编辑窗口都显示空态或现有失败状态，`MainViewModel` 继续记录日志。
- 如果封面预览生成失败，沿用现有 `generateCoverPreview` 的日志和 `coverPreview = null` 行为。
- 如果没有选择封面，编辑窗口显示选择封面的入口，不允许用户误以为已有可合成封面。

## 测试与验证

实现后至少验证：

- `./gradlew compileKotlin` 通过。
- 主窗口点击“编辑封面”能打开独立窗口。
- 重复点击不会打开多个封面编辑窗口。
- 关闭编辑窗口不会清空封面设置。
- 在编辑窗口选择图片，主窗口封面缩略图和预览同步更新。
- 在编辑窗口切换背景模式和自定义颜色，主窗口预览同步更新。
- 合成中封面编辑控件禁用。
- 原有合成流程仍使用最新封面设置生成 `cover.png`。
