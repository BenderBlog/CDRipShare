我想写一个GUI工具，要求输入若干音频和一个图片，从而生成一个视频。
输入：
 - 若干个*相同编码*的音频文件，无损编码包括 FLAC、ALAC、APE，有损编码包括 aac 或 mp3
 - 一张图片
状态：
 - 待命状态，进度条空，输入允许：GUI 中输入音频文件、图片文件一个，点击提交
 - 工作状态，进度条非空且显示进展，输入不允许：后台进行合成流程，在流程章节说明
 - 报错状态，进度条非空且飘红，输入不允许：显示报错详情，要求用户重启工作或开启新输入
流程：
 - 按照用户输入的音频文件顺序，无损拼接文件
 - 将图片和音频合成一个视频，其中
    - 音频流编码要求和输入的音频编码一样
    - 视频流可以编码成 h265，按照 stillimage 或类似的 profile 编码
相关其他
 - 使用 Jetpack Compose 技术，不考虑移动端平台
 - 使用 FFMPEG 技术
GUI 大致样式
Scaffold
    - Appbar
        - Title
    - Body
        - List：选择的音频文件列表，可以重新排序
        - 添加和删除按钮，用于操控音频文件
        - 图片选择工具
        - 开始合成按钮
        - 进度条
        - 日志输出和报错输出
